# ⚽ Real Madrid OOP Simulator (C++)

Цей міні-проєкт є чудовою демонстрацією базових принципів **Об'єктно-орієнтованого програмування (ООП)** мовою C++ на прикладі найутитулованішого футбольного клубу світу — "Реал Мадрид" (Real Madrid). 

Проєкт показує, як працює **інкапсуляція** (приховування даних) та **наслідування** (створення нових класів на базі існуючих).

## 🏗 Архітектура класів

Проєкт складається з двох взаємопов'язаних класів:

### 1. `RealMadridClub` (Базовий клас)
Відповідає за фундаментальну, історичну інформацію про клуб.
* **Приватні поля (Encapsulation):**
  * `president` — ім'я поточного президента клубу.
  * `headCoach` — ім'я головного тренера.
  * `uclTrophies` — кількість здобутих кубків Ліги Чемпіонів.
* **Публічні методи:**
  * `setManagement()` — встановлює президента та тренера.
  * `winChampionsLeague()` — збільшує лічильник трофеїв ЛЧ на 1 🏆.
  * `getTrophies()` — повертає поточну кількість кубків.
  * `printClubInfo()` — виводить загальну інформацію про клуб на екран.

### 2. `CurrentSquad` (Похідний клас)
Наслідує всі властивості та методи базового класу `RealMadridClub` і доповнює їх інформацією про поточний ігровий сезон.
* **Приватні поля:**
  * `mainStar` — головна зірка поточного складу.
  * `squadValue` — загальна трансферна вартість складу (в млн євро).
* **Публічні методи:**
  * `setSquadDetails()` — задає зірку та вартість команди.
  * `printSeasonStats()` — виводить детальну статистику поточного складу, включаючи базову інформацію клубу.

## 🧠 Які концепти C++ тут використано?

* **Inheritance (Наслідування):** Клас `CurrentSquad` автоматично отримує доступ до публічних методів `RealMadridClub` (наприклад, може викликати `winChampionsLeague()`).
* **Encapsulation (Інкапсуляція):** Всі змінні заховані в секціях `private`, що захищає їх від випадкової зміни ззовні. Доступ до них здійснюється лише через спеціальні публічні методи (геттери/сеттери).
* **Конструктори:** Класи мають власні конструктори для правильної початкової ініціалізації об'єктів.
* **Константні методи (`const`):** Методи виводу даних позначені як `const`, що гарантує незмінність даних під час їх читання.
